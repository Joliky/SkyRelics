# SkyRelics: аудит уязвимостей и нагрузки

## Что было найдено

### Обход protection-плагинов

Массовые способности вроде `VEIN_MINE`, `TREE_FELLER`, `TUNNEL_BREAK`, `DIG_3X3`, `HARVEST_RADIUS` меняли дополнительные блоки напрямую. Это могло обходить плагины защиты территорий, потому что они видели только исходный `BlockBreakEvent`.

Исправлено:

- дополнительные сломанные блоки проходят через отдельный `BlockBreakEvent`;
- изменение блоков для `PATH_MAKER`, `SOIL_BLESSING`, `EARTH_WALL` проходит через `BlockPlaceEvent`;
- блоки с инвентарем, bedrock, barrier, command blocks и end portal frame не ломаются массовыми способностями.

### Работа по отмененным событиям

Способности могли выполняться даже если другое расширение отменило событие.

Исправлено:

- обработчики используют `ignoreCancelled = true`.

### Рост памяти

Кулдауны, метки целей, combo/rage-состояния могли оставаться в памяти после истечения.

Исправлено:

- `CooldownManager` очищает истекшие записи;
- listener раз в минуту чистит истекшие mark/combo/rage-состояния.

### Лишняя нагрузка на частых событиях

Определение предмета по display name каждый раз сканировало все `items` в конфиге.

Исправлено:

- `NamespacedKey` создаются один раз;
- fallback-поиск по display name использует кеш, который обновляется не чаще раза в секунду.

### Неудачная логика кулдаунов

Если одна способность в списке была на кулдауне, следующие способности этого же триггера не выполнялись.

Исправлено:

- способность на кулдауне пропускается, остальные продолжают выполняться.

### Дублирование урожая

`HARVEST_RADIUS` и `REPLANT` могли конфликтовать с обычным ломанием блока: дроп мог дублироваться, а исходная культура не пересаживалась.

Исправлено:

- исходный урожай обрабатывается вручную;
- обычный break отменяется для корректного сбора и пересадки.

### Потеря предметов при выдаче

Если инвентарь игрока был заполнен, остатки от `/sr give` могли быть потеряны.

Исправлено:

- остатки предметов выпадают рядом с игроком;
- количество в команде ограничено диапазоном `1..64`.

## Ограничения нагрузки

В код добавлены верхние лимиты:

| Лимит | Значение |
|---|---:|
| Максимальный радиус area-способностей | `4` |
| Максимальный радиус поиска сущностей/дропа | `12` |
| Максимум дополнительных блоков за одно массовое ломание | `128` |
| Максимум целей в цепной молнии | `8` |
| Максимум временных блоков стены | `64` |
| Максимальная длительность временной стены | `30 секунд` |
| Максимум волн `LIFE_BLOOM` | `10` |

## Рекомендации для сервера

- Не ставить маленькие кулдауны на `VEIN_MINE`, `TREE_FELLER`, `DIG_3X3`, `CHAIN_LIGHTNING`, `AOE_SWEEP`.
- Для публичного сервера держать `max-blocks` в диапазоне `16..64`.
- Для `EARTH_WALL` использовать небольшие размеры: `width: 3`, `height: 2`.
- Не выдавать несколько массовых инструментов всем игрокам без проверки TPS на реальном онлайне.
- После изменения `config.yml` использовать `/sr reload`, а для изменения названия/lore/NBT лучше выдать предмет заново.
