# SkyRelics: документация по RPG-способностям

Документ описывает систему предметов и способностей SkyRelics для Minecraft 1.16.5.

## Кратко о плагине

SkyRelics позволяет создавать кастомные предметы в `config.yml` и выдавать их игрокам командой:

```text
/sr give <ник> <item_id> [количество]
```

Полная команда:

```text
/skyrelics give <ник> <item_id> [количество]
```

Дополнительные команды:

```text
/sr list [page]
/sr info <item_id>
/sr inspect
/sr validate
/sr abilities [page]
/sr giveall <item_id> [количество]
/sr cooldown clear <ник> [key|all]
/sr reload
/sr help
```

Tab-complete подставляет доступные подкоманды, онлайн-игроков, `item_id`, страницы списка, количество предметов и аргументы очистки кулдаунов с учетом прав игрока.

## Ограничения предметов

У любого предмета можно указать условия, где его способности работают. Проверка выполняется перед запуском любого триггера способности.

```yaml
items:
  dungeon_pickaxe:
    material: DIAMOND_PICKAXE
    name: "&bDungeon Pickaxe"
    worlds:
      - dungeon
    blocked-worlds:
      - world_nether
    gamemodes:
      - SURVIVAL
      - ADVENTURE
    biomes:
      - PLAINS
      - FOREST
    min-y: 5
    max-y: 80
    permission: "skyrelics.use.dungeon_pickaxe"
    bind-on-first-use: true
    owner-only: true
    bind-on-give: false
```

| Поле | Что делает |
|---|---|
| `worlds` | Белый список миров. Если список пустой, разрешены все миры |
| `blocked-worlds` | Черный список миров |
| `gamemodes` | Разрешенные режимы игры: `SURVIVAL`, `ADVENTURE`, `CREATIVE`, `SPECTATOR` |
| `biomes` | Белый список биомов Bukkit, например `PLAINS`, `FOREST`, `DESERT` |
| `min-y` | Минимальная высота использования |
| `max-y` | Максимальная высота использования |
| `permission` | Permission, без которого предмет не запускает способности |
| `bind-on-first-use` | При первом успешном использовании записывает владельца в NBT предмета |
| `owner-only` | Если владелец записан, способности работают только у него |
| `bind-on-give` | Привязывает предмет к игроку сразу при выдаче через `/sr give` или `/sr giveall` |

Если ограничение не пройдено, способность не запускается, а игрок получает короткое сообщение. Тексты можно переопределить в `messages` ключами `item-restricted-world`, `item-restricted-gamemode`, `item-restricted-biome`, `item-restricted-height`, `item-restricted-permission`.

Для редких RPG-предметов обычно удобно ставить:

```yaml
bind-on-first-use: true
owner-only: true
```

Для донатных или квестовых предметов, которые не должны передаваться даже до первого использования:

```yaml
bind-on-give: true
owner-only: true
```

Каждый предмет задается в секции `items`. У предмета есть материал, название, описание, зачарования и список способностей.

Пример:

```yaml
items:
  rpg_sword:
    material: DIAMOND_SWORD
    name: "&d&lRPG Sword"
    lore:
      - "&7A complete sword relic with elemental and combo powers."
    unbreakable: true
    enchantments:
      DAMAGE_ALL: 4
      DURABILITY: 3
    abilities:
      attack-entity:
        - type: FIRE_SLASH
          chance: 0.18
          fire-ticks: 100
```

## Триггеры способностей

Триггер определяет, когда способность срабатывает.

| Триггер | Когда срабатывает | Для чего использовать |
|---|---|---|
| `right-click` | Игрок нажал ПКМ предметом в руке | Активные умения, рывки, зоны, баффы |
| `shift-right-click` | Игрок нажал Shift + ПКМ | Переключение режимов, особые активные умения |
| `break-block` | Игрок сломал блок предметом | Кирки, топоры, лопаты, мотыги |
| `attack-entity` | Игрок ударил живую цель | Мечи, топоры, боевые инструменты |
| `kill-entity` | Игрок убил существо | Баффы за убийство, накопление душ |
| `receive-damage` | Игрок получил урон с предметом в руке | Шипы, защитные реакции, контратаки |

## Общие параметры способностей

Эти параметры можно использовать у большинства способностей.

| Параметр | Тип | Значение |
|---|---|---|
| `type` | строка | Название способности. Обязательный параметр |
| `cooldown` | число | Кулдаун в миллисекундах |
| `chance` | число | Шанс срабатывания. Можно писать `0.25` или `25` |
| `show-cooldown` | boolean | Показывать ли сообщение о кулдауне. По умолчанию `true` |
| `message` | строка | Сообщение игроку, если способность его поддерживает |

Пример с шансом и кулдауном:

```yaml
- type: CHAIN_LIGHTNING
  chance: 0.25
  cooldown: 12000
  chains: 3
  radius: 6
  damage: 3.0
```

## Базовые способности

### `TOGGLE_ENCHANT`

Переключает предмет между двумя зачарованиями.

Лучший триггер: `shift-right-click`.

Параметры:

| Параметр | Значение |
|---|---|
| `first-enchant` | Первое зачарование |
| `first-level` | Уровень первого зачарования |
| `second-enchant` | Второе зачарование |
| `second-level` | Уровень второго зачарования |
| `first-message` | Сообщение при переключении на первое |
| `second-message` | Сообщение при переключении на второе |

Пример:

```yaml
- type: TOGGLE_ENCHANT
  first-enchant: LOOT_BONUS_BLOCKS
  first-level: 3
  second-enchant: SILK_TOUCH
  second-level: 1
```

### `POTION_EFFECTS`

Выдает игроку один или несколько эффектов зелий.

Лучший триггер: `right-click`, `kill-entity`, `break-block`.

Параметры:

| Параметр | Значение |
|---|---|
| `effects` | Список эффектов |
| `message` | Сообщение игроку |

Пример:

```yaml
- type: POTION_EFFECTS
  effects:
    - type: SPEED
      duration: 100
      amplifier: 1
```

### `TELEPORT_TARGET`

Телепортирует игрока к блоку, на который он смотрит.

Лучший триггер: `right-click`.

Параметры:

| Параметр | Значение |
|---|---|
| `distance` | Максимальная дистанция поиска блока |

### `DASH`

Толкает игрока вперед.

Лучший триггер: `right-click`.

Параметры:

| Параметр | Значение |
|---|---|
| `power` | Сила рывка вперед |
| `y` | Вертикальная сила |
| `message` | Сообщение игроку |

### `HEAL`

Восстанавливает здоровье и/или сытость.

Лучший триггер: `right-click`, `kill-entity`.

Параметры:

| Параметр | Значение |
|---|---|
| `health` | Сколько здоровья восстановить |
| `food` | Сколько сытости восстановить |
| `message` | Сообщение игроку |

### `LIGHTNING_TARGET`

Создает молнию в блоке, на который смотрит игрок.

Лучший триггер: `right-click`.

Параметры:

| Параметр | Значение |
|---|---|
| `distance` | Дистанция поиска блока |
| `damage` | `true` - настоящая молния с уроном, `false` - визуальный эффект |
| `message` | Сообщение игроку |

### `PARTICLE`

Создает частицы около игрока.

Параметры:

| Параметр | Значение |
|---|---|
| `particle` | Название частицы Bukkit |
| `count` | Количество частиц |
| `y-offset` | Смещение вверх |
| `offset-x` | Разброс по X |
| `offset-y` | Разброс по Y |
| `offset-z` | Разброс по Z |
| `speed` | Скорость частиц |

### `SOUND`

Проигрывает звук около игрока.

Параметры:

| Параметр | Значение |
|---|---|
| `sound` | Название звука Bukkit |
| `volume` | Громкость |
| `pitch` | Тон |

### `MESSAGE`

Отправляет игроку сообщение.

Параметры:

| Параметр | Значение |
|---|---|
| `message` | Текст сообщения |

## Способности кирки

### `VEIN_MINE`

Ломает жилу руды рядом с исходным блоком.

Триггер: `break-block`.

Работает только с рудами:

`COAL_ORE`, `IRON_ORE`, `GOLD_ORE`, `REDSTONE_ORE`, `LAPIS_ORE`, `DIAMOND_ORE`, `EMERALD_ORE`, `NETHER_QUARTZ_ORE`.

Параметры:

| Параметр | Значение |
|---|---|
| `max-blocks` | Максимум дополнительных блоков |

Пример:

```yaml
- type: VEIN_MINE
  max-blocks: 18
```

### `AUTO_SMELT`

Заменяет обычный дроп переплавляемого блока на готовый результат.

Триггер: `break-block`.

Поддерживаемые блоки:

| Блок | Дроп |
|---|---|
| `IRON_ORE` | `IRON_INGOT` |
| `GOLD_ORE` | `GOLD_INGOT` |
| `SAND` | `GLASS` |
| `COBBLESTONE` | `STONE` |

Параметры:

| Параметр | Значение |
|---|---|
| `amount` | Количество предметов. По умолчанию `1` |

### `ORE_MAGNET`

Телепортирует выпавшие предметы рядом к игроку.

Триггер: `break-block`.

Параметры:

| Параметр | Значение |
|---|---|
| `radius` | Радиус поиска дропа |

### `MINING_BURST`

Выдает игроку эффект быстрого копания.

Триггер: `break-block`.

Параметры:

| Параметр | Значение |
|---|---|
| `duration` | Длительность в тиках |
| `amplifier` | Уровень эффекта |

### `TUNNEL_BREAK`

Ломает дополнительные блоки вокруг сломанного блока.

Триггер: `break-block`.

То же ядро используется для `DIG_3X3` и `AREA_BREAK`.

Параметры:

| Параметр | Значение |
|---|---|
| `radius` | Радиус области |
| `max-blocks` | Максимум дополнительных блоков |
| `same-type-only` | Ломать только блоки того же типа |

### `TREASURE_CHANCE`

С шансом выбрасывает дополнительный предмет.

Триггер: `break-block`.

Параметры:

| Параметр | Значение |
|---|---|
| `chance` | Шанс срабатывания |
| `drop` | Материал дополнительного дропа |
| `amount` | Количество |

### `EXPLOSIVE_MINE`

Создает небольшой взрыв на месте блока.

Триггер: `break-block`.

Параметры:

| Параметр | Значение |
|---|---|
| `power` | Сила взрыва |
| `break-blocks` | Ломать ли блоки взрывом |

Для RPG-сервера обычно лучше ставить `break-blocks: false`.

### `FORTUNE_SURGE`

С шансом дает дополнительный дроп руды.

Триггер: `break-block`.

Параметры:

| Параметр | Значение |
|---|---|
| `bonus` | Количество бонусного дропа |
| `chance` | Шанс срабатывания |

### `REPAIR_ON_ORE`

Чинит предмет при добыче руды.

Триггер: `break-block`.

Параметры:

| Параметр | Значение |
|---|---|
| `repair` | Сколько прочности восстановить |

### `STONE_SKIN`

Выдает игроку сопротивление урону.

Триггер: `break-block`.

Параметры:

| Параметр | Значение |
|---|---|
| `duration` | Длительность в тиках |
| `amplifier` | Уровень эффекта |

## Способности топора

### `TREE_FELLER`

Срубает дерево целиком, ломая связанные бревна.

Триггер: `break-block`.

Параметры:

| Параметр | Значение |
|---|---|
| `max-blocks` | Максимум бревен |

### `BLEEDING_STRIKE`

Накладывает кровотечение: цель получает периодический урон.

Триггер: `attack-entity`.

Параметры:

| Параметр | Значение |
|---|---|
| `ticks` | Количество ударов кровотечения, раз в секунду |
| `damage` | Урон за один тик кровотечения |

### `ARMOR_CRACK`

Ослабляет цель эффектом `WEAKNESS`.

Триггер: `attack-entity`.

Параметры:

| Параметр | Значение |
|---|---|
| `duration` | Длительность в тиках |
| `amplifier` | Уровень слабости |

### `EXECUTE`

Увеличивает урон по целям с низким здоровьем.

Триггер: `attack-entity`.

Параметры:

| Параметр | Значение |
|---|---|
| `health-percent` | Порог здоровья цели. Например `0.25` значит 25% |
| `multiplier` | Множитель урона |

### `BATTLE_ROAR`

Отталкивает врагов рядом, ослабляет их и выдает игроку `STRENGTH`.

Триггер: `right-click`.

Параметры:

| Параметр | Значение |
|---|---|
| `radius` | Радиус действия |
| `power` | Сила отталкивания |
| `duration` | Длительность эффектов |
| `amplifier` | Уровень силы игрока |

### `LOG_SPLINTER`

С шансом дает дополнительный дроп при рубке дерева.

Триггер: `break-block`.

Параметры такие же, как у `TREASURE_CHANCE`.

### `RAGE_STACKS`

Каждый удар накапливает стаки ярости и добавляет бонусный урон.

Триггер: `attack-entity`.

Параметры:

| Параметр | Значение |
|---|---|
| `max-stacks` | Максимум стаков |
| `damage-per-stack` | Дополнительный урон за стак |
| `keep-ms` | Сколько миллисекунд стаки живут без нового удара |

### `SHIELD_BREAKER`

Усиливает урон по игроку, который блокирует щитом, и ставит щит на кулдаун.

Триггер: `attack-entity`.

Параметры:

| Параметр | Значение |
|---|---|
| `multiplier` | Множитель урона |
| `shield-cooldown` | Кулдаун щита в тиках |

### `WILD_CHARGE`

Толкает игрока к цели и усиливает удар.

Триггер: `attack-entity`.

Параметры:

| Параметр | Значение |
|---|---|
| `power` | Сила рывка |
| `y` | Вертикальная сила |
| `damage-multiplier` | Множитель урона |

### `LIFESTEAL_CHOP`

Лечит игрока на часть нанесенного урона.

Триггер: `attack-entity`.

Параметры:

| Параметр | Значение |
|---|---|
| `percent` | Доля урона, которая вернется здоровьем |

## Способности мотыги

### `HARVEST_RADIUS`

Собирает созревшие культуры в радиусе.

Триггер: `break-block`.

Параметры:

| Параметр | Значение |
|---|---|
| `radius` | Радиус сбора |

Сбрасывает возраст культуры на 0, то есть пересаживает ее.

Триггер: `break-block`.

### `GROWTH_AURA`

Ускоряет рост культур рядом с игроком.

Триггер: `right-click`.

Параметры:

| Параметр | Значение |
|---|---|
| `radius` | Радиус действия |
| `growth` | На сколько стадий увеличить возраст культуры |

### `NATURE_HEAL`

Лечит игрока, если рядом есть растения, листья или цветы.

Триггер: `right-click`.

Параметры:

| Параметр | Значение |
|---|---|
| `radius` | Радиус проверки растений |
| `health` | Сколько здоровья восстановить |

### `POISON_THORNS`

Отравляет атакующего, когда игрок получает урон.

Триггер: `receive-damage`.

Параметры:

| Параметр | Значение |
|---|---|
| `duration` | Длительность яда |
| `amplifier` | Уровень яда |

### `FARMER_LUCK`

С шансом дает дополнительный дроп при сборе.

Триггер: `break-block`.

### `COMPOST_TOUCH`

С шансом дает дополнительный предмет, обычно костную муку.

Параметры:

| Параметр | Значение |
|---|---|
| `distance` | Дистанция поиска цели по взгляду |

Выдает игроку сопротивление урону.

Триггер: `right-click`.

Параметры такие же, как у `STONE_SKIN`.

### `LIFE_BLOOM`

Создает несколько волн лечения для игроков рядом.

Триггер: `right-click`.

Параметры:

| Параметр | Значение |

| `radius` | Радиус лечения |
| `pulses` | Количество волн |
| `health` | Лечение за одну волну |

## Способности меча

### `FIRE_SLASH`

Поджигает цель.

Триггер: `attack-entity`.

Параметры:

| Параметр | Значение |
|---|---|
| `fire-ticks` | Время горения в тиках |

### `ICE_SLASH`

Замедляет цель и показывает снежные частицы.

Триггер: `attack-entity`.

Параметры:

| Параметр | Значение |
|---|---|
| `duration` | Длительность замедления |
| `amplifier` | Сила замедления |

### `CHAIN_LIGHTNING`

Бьет цель молнией и перекидывается на ближайших существ.

Триггер: `attack-entity`.

Параметры:

| Параметр | Значение |
|---|---|
| `chains` | Максимум целей в цепи |
| `radius` | Радиус поиска следующей цели |
| `damage` | Урон каждой цели |

### `VAMPIRIC_HIT`

Лечит игрока на часть нанесенного урона.

Триггер: `attack-entity`.

Параметры такие же, как у `LIFESTEAL_CHOP`.

### `SOUL_STEAL`

После убийства выдает игроку силу и регенерацию.

Триггер: `kill-entity`.

Параметры:

| Параметр | Значение |
|---|---|
| `duration` | Длительность эффектов |
| `amplifier` | Уровень силы |

### `CRITICAL_DASH`

Рывок к цели с усилением удара.

Триггер: `attack-entity`.

Параметры такие же, как у `WILD_CHARGE`.

### `AOE_SWEEP`

Наносит урон существам рядом с целью.

Триггер: `attack-entity`.

Параметры:

| Параметр | Значение |
|---|---|
| `radius` | Радиус урона |
| `percent` | Доля исходного урона |
| `damage` | Фиксированный урон, если исходного события нет |

### `FEAR`

Ослабляет цель, замедляет ее, отталкивает от игрока и сбрасывает цель у мобов.

Триггер: `attack-entity`.

Параметры:

| Параметр | Значение |
|---|---|
| `duration` | Длительность слабости и замедления |
| `power` | Сила отталкивания |

### `MARK_TARGET`

Помечает цель. Пока метка активна, цель получает увеличенный урон.

Триггер: `attack-entity`.

Параметры:

| Параметр | Значение |
|---|---|
| `multiplier` | Множитель входящего урона по цели |
| `duration-ms` | Длительность метки в миллисекундах |

### `COMBO_STRIKE`

Считает серию ударов. Каждый N-й удар усиливается.

Триггер: `attack-entity`.

Параметры:

| Параметр | Значение |
|---|---|
| `trigger` | На каком ударе сработает комбо |
| `multiplier` | Множитель урона |
| `keep-ms` | Сколько миллисекунд серия живет без нового удара |

## Рекомендации по балансу

Для RPG-сервера лучше не ставить слишком много сильных способностей без кулдаунов. Особенно это касается:

| Способность | Что ограничивать |
|---|---|
| `TUNNEL_BREAK`, `DIG_3X3` | `cooldown`, `max-blocks` |
| `VEIN_MINE`, `TREE_FELLER` | `max-blocks` |
| `CHAIN_LIGHTNING`, `AOE_SWEEP` | `cooldown`, `damage`, `radius` |
| `LIFE_BLOOM`, `NATURE_HEAL`, `VAMPIRIC_HIT` | `cooldown`, `health`, `percent` |
| `EARTH_WALL` | `cooldown`, `duration`, `width`, `height` |

Практичные стартовые значения:

```yaml
cooldown: 8000      # 8 секунд
chance: 0.15        # 15%
radius: 3
duration: 100       # 5 секунд
```

## Как добавить новую способность предмету

1. Открой `src/main/resources/config.yml`.
2. Найди нужный предмет в `items`.
3. Добавь способность в подходящий триггер.
4. Выполни `/sr reload` на сервере.
5. Если предмет уже выдан, лучше выдать его заново через `/sr give`, чтобы обновить название, lore или NBT-id.

Пример:

```yaml
items:
  my_sword:
    material: DIAMOND_SWORD
    name: "&cBlade of Storms"
    lore:
      - "&7Every hit can call lightning."
    abilities:
      attack-entity:
        - type: CHAIN_LIGHTNING
          chance: 0.2
          cooldown: 10000
          chains: 3
          radius: 6
          damage: 3.0
```
